git revert -m 1(or2)

当你在B分支上把A merge到B中, 那么B就是merge commit的parent1, 而A是parent2.

由于merge commit 是将两条线合并到一条线上, 因此, 合并时的那个commit, 将具有两个祖先. 所以git不知道base是选择哪个parent进行diff, 所以就抱怨了, 所以你要用-m属性显示地告诉git用哪一个parent.
